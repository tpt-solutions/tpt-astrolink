# S3 Object Layout — FITS Storage

**Owner:** TPT Cloud Core (Go) / TPT Edge Agent (Rust) · **Spec ref:** Data flow "Data".

## Buckets

| Bucket | Purpose | Lifecycle |
|--------|---------|-----------|
| `tpt-astrolink-fits` | Raw & processed FITS images | Raw `.fits.gz` → transition to Glacier after 90d; keep index. |
| `tpt-astrolink-thumbs` | PNG/JPG preview thumbnails for UI | Expire after 365d. |
| `tpt-astrolink-models` | Edge AI ONNX model artifacts | Versioned, no expiry. |

## Object Key Patterns

### Raw FITS (uploaded by Edge Agent)

```
fits/raw/<nodeId>/<yyyy>/<mm>/<dd>/<observationId>/<frame>-<seq>.fits.gz
```

- `<nodeId>` — Edge Node UUID (no PII).
- Compressed with gzip (`Content-Type: application/x-gzip-fits`).
- Metadata set: `node-id`, `observation-id`, `frame`, `captured-at` (RFC3339).

### Astrometry output (written by Cloud worker)

```
fits/solved/<nodeId>/<yyyy>/<mm>/<dd>/<observationId>/<frame>-<seq>.fits
fits/wcs/<observationId>/<frame>.wcs            # world-coordinate system sidecar
```

### Thumbnails (generated by Cloud worker)

```
thumbs/<observationId>/<frame>.jpg
```

### Edge AI model artifacts

```
models/transient/<version>/brightness-anomaly.onnx
```

## Access

- Edge Agent: `PutObject` on `fits/raw/<its-node-id>/*` only (per-node IAM/regex policy).
- Cloud Core worker: `GetObject`/`PutObject` on `fits/*`, `thumbs/*`.
- Client UI: no direct S3 access; presigned URLs generated by Cloud Core.

## Cross-reference

- Postgres `metadata.object_key` stores the S3 key of the solved FITS.
